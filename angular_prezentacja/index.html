Troche teorii

Bez Angularjs

  • event loop przeglądarki czeka na zdarzenie
  • pojawia się zdarzenie (wpisujemy tekst)
  • event callback jest wywoływany i JS modyfikuję DOM
  • gdy callback jest wykonany przeglądarka wyświetla widok na podstawie DOM

Troche teorii

Angularjs

  • własny flow reagowania na zmiany, który dzieli się na zdarzenia w kontekście Angular i poza
  • pojawia się zdarzenie (wpisujemy tekst)
  • odpalany jest $digest loop, który składa się z dwóch mniejszych pętli
  • $evalAsync (co wykonać) i $watch (co zmienić)
  • gdy pętla $digest jest zakończona przeglądarka robi re-render DOM

Modules and Dependency Injection

"I was expecting a paradigm shift, and all I got was a lousy constructor"

DI in AngularJS

  • $provide allows us to register recipes for object creation
  • $injector gives access to singletons produced based on those recipes (service locator)
  • Singleton instances are called "services"

Bezpośrednie wstrzykiwanie

            
                function MyController($scope, greeter) {
                // ...
                }

Przez $injector

            
                var MyController = function(renamed$scope, renamedGreeter) {
                ...
                }
                MyController['$inject'] = ['$scope', 'greeter'];

Rekomendowane

            
                someModule.factory('greeter', ['$window', function(renamed$window) {
                // ...
                }]);

Modules accept various recipes

  • Values
  • Services
  • Factories
  • Constants
